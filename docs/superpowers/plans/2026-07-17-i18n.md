# i18n (pt-BR / en-US) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a PT/EN locale toggle across the whole site (`/` + `/lab`), with browser-default language and localStorage after explicit choice — no new i18n libraries.

**Architecture:** Mirror `ThemeProvider`: `LocaleProvider` + `useLocale` + `LocaleToggle`. Copy lives in typed dictionaries (`pt-BR.ts` / `en-US.ts`); components call `useT()`. Contact identity (name, email, URLs) stays in `sharedPitch.ts` untranslated.

**Tech Stack:** React 19, Vite 6, TypeScript, existing Tailwind/Headless UI patterns. No i18next.

**Spec:** `docs/superpowers/specs/2026-07-17-i18n-design.md`

## Global Constraints

- Locales only: `pt-BR` | `en-US`
- First visit: `navigator.language` starts with `en` → `en-US`, else `pt-BR`; do **not** write `localStorage` until user toggles
- After toggle: persist key `locale` in `localStorage`
- Set `document.documentElement.lang` to the active locale
- No locale in URL; no IP/geo; no new npm i18n deps
- Do not translate: proper name, email, URLs, handles, lib/sigla names (`React`, `n8n`, `RAG`, …), DOM ids
- No test framework — verify with `npm run build` (+ `npm run lint` when touching UI)
- **Do not commit** unless the user explicitly asks

- [ ] **Step 4: Create `src/i18n/index.ts`**

```ts
import type { Locale } from '../context/locale';
import { enUS } from './en-US';
import { ptBR, type Messages } from './pt-BR';

export type { Messages };
export type MessageKey = keyof Messages;

export const messages: Record<Locale, Messages> = {
  'pt-BR': ptBR,
  'en-US': enUS,
};

export function resolveLocaleFromNavigator(
  language = typeof navigator !== 'undefined' ? navigator.language : 'pt-BR',
): Locale {
  return language.toLowerCase().startsWith('en') ? 'en-US' : 'pt-BR';
}
```

- [ ] **Step 5: Create `src/context/LocaleContext.tsx`**

```tsx
import { useEffect, useState } from 'react';
import { LocaleContext, type Locale } from './locale';
import { resolveLocaleFromNavigator } from '../i18n';

const STORAGE_KEY = 'locale';

function readStoredLocale(): Locale | null {
  const stored = localStorage.getItem(STORAGE_KEY);
  if (stored === 'pt-BR' || stored === 'en-US') return stored;
  return null;
}

export function LocaleProvider({ children }: { children: React.ReactNode }) {
  const [locale, setLocaleState] = useState<Locale>(() => {
    return readStoredLocale() ?? resolveLocaleFromNavigator();
  });

  useEffect(() => {
    document.documentElement.lang = locale;
  }, [locale]);

  const setLocale = (next: Locale) => {
    setLocaleState(next);
    localStorage.setItem(STORAGE_KEY, next);
  };

  const toggleLocale = () => {
    setLocale(locale === 'pt-BR' ? 'en-US' : 'pt-BR');
  };

  return (
    <LocaleContext.Provider value={{ locale, setLocale, toggleLocale }}>
      {children}
    </LocaleContext.Provider>
  );
}
```

- [ ] **Step 6: Create `src/context/useLocale.ts`**

```ts
import { useContext } from 'react';
import { LocaleContext } from './locale';

export function useLocale() {
  const ctx = useContext(LocaleContext);
  if (!ctx) {
    throw new Error('useLocale must be used within a LocaleProvider');
  }
  return ctx;
}
```

- [ ] **Step 7: Create `src/i18n/useT.ts`**

```ts
import { useLocale } from '../context/useLocale';
import { messages, type MessageKey } from './index';

export function useT() {
  const { locale } = useLocale();
  return (key: MessageKey) => messages[locale][key];
}
```

- [ ] **Step 1: Create `LocaleToggle`**

Segmented PT | EN control (not flags). Active locale gets stronger weight / ink; inactive muted. Clicking the inactive side (or toggle) calls `setLocale`.

```tsx
import { useLocale } from '../context/useLocale';
import { useT } from '../i18n/useT';
import type { Locale } from '../context/locale';

interface LocaleToggleProps {
  className?: string;
}

export default function LocaleToggle({ className }: LocaleToggleProps) {
  const { locale, setLocale } = useLocale();
  const t = useT();

  const btn = (code: Locale, label: string) => (
    <button
      type="button"
      onClick={() => setLocale(code)}
      aria-pressed={locale === code}
      className={`px-2 py-1 font-mono text-xs uppercase tracking-wider transition ${
        locale === code
          ? 'font-bold opacity-100'
          : 'opacity-45 hover:opacity-80'
      }`}
    >
      {label}
    </button>
  );

  return (
    <div
      className={className}
      role="group"
      aria-label={t('locale.toggle')}
    >
      <div className="inline-flex items-center rounded-full bg-gray-600/80 px-1 py-0.5 text-cream dark:bg-gray-400/30">
        {btn('pt-BR', 'PT')}
        <span className="opacity-30" aria-hidden>
          |
        </span>
        {btn('en-US', 'EN')}
      </div>
    </div>
  );
}
```

On `/lab`, reuse the same component; if gray/cream clashes with Memphis, wrap with a lab-friendly class from the parent (e.g. border + `--mp-ink` / `--mp-surface`) — keep one component, style via `className` overrides on the inner wrapper if needed. Prefer a neutral style that works on both pages: `border border-current/30 bg-black/10 px-1` without cream dependency if lab looks wrong.

Recommended dual-safe markup for the inner pill:

```tsx
<div className="inline-flex items-center border border-current/25 bg-black/5 px-1 py-0.5 backdrop-blur-sm dark:bg-white/10">
```
