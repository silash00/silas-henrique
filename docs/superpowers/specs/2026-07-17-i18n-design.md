# i18n (pt-BR / en-US) — Design Spec

**Date:** 2026-07-17  
**Status:** Approved  
**Scope:** Site inteiro (`/` + `/lab`)

## Goal

Permitir alternar o conteúdo textual do site entre **português (pt-BR)** e **inglês (en-US)** via toggle, no mesmo espírito do theme toggle — sem detecção por IP e sem locale na URL.

## Decisions (locked)

| Topic | Choice |
|-------|--------|
| Cobertura | Site inteiro (root + lab) |
| Locales | `pt-BR` \| `en-US` |
| Detecção inicial | `navigator.language`: se começa com `en` → `en-US`; senão → `pt-BR` |
| Persistência | `localStorage` key `locale` (após primeira escolha explícita **ou** após resolução inicial — ver nota abaixo) |
| URL | Sem locale na rota / query |
| Geo / IP | Fora de escopo |
| Stack | Context + dicionários TypeScript (sem i18next / react-intl) |
| `document.documentElement.lang` | Atualizado para `pt-BR` ou `en-US` |

### Persistência — comportamento

Espelhar o tema:

1. Se `localStorage.locale` existe → usar esse valor.
2. Senão → derivar de `navigator.language` e **não** gravar até o usuário clicar no toggle (igual `darkMode` só grava no toggle). Assim mudanças futuras de idioma do browser ainda podem refletir se o usuário nunca escolheu.
3. No toggle → gravar a escolha em `localStorage`.

## Architecture

### Provider

```
src/context/
  locale.ts          # Locale type + Context
  LocaleContext.tsx  # LocaleProvider
  useLocale.ts       # hook (locale, setLocale, toggleLocale)
```

API pública:

```ts
type Locale = 'pt-BR' | 'en-US';

type LocaleContextType = {
  locale: Locale;
  setLocale: (locale: Locale) => void;
  toggleLocale: () => void;
};
```

`LocaleProvider` envolve a app em `App.tsx` junto com `ThemeProvider`.

### Dicionários

```
src/i18n/
  types.ts       # Messages type (estrutura tipada)
  pt-BR.ts
  en-US.ts
  index.ts       # messages map + helper resolveLocaleFromNavigator()
  useT.ts        # t(key) com autocomplete via nested keys ou flat keys
```

**Formato preferido:** chaves flat com namespaces por ponto, tipadas:

```ts
// exemplo
{
  'root.hello': 'Olá!',
  'root.intro': '...',
  'lab.hero.headline': '...',
  'lab.hero.cta.works': 'Ver trabalhos',
  'lab.hero.cta.contact': 'Falar comigo',
  // ...
}
```

`useT()` retorna `(key: MessageKey) => string`. Sem interpolação avançada no MVP; se precisar de um pedaço variável raro, concatenar no componente ou aceitar `{name}` simples depois.

### Toggle UI

Novo componente `LocaleToggle` (espelha posição/estilo do `ThemeToggle` o suficiente para parecer da mesma família):

- Controle segmentado ou switch com labels **PT** / **EN** (não bandeiras).
- `aria-label` localizado (ex.: "Idioma" / "Language").
- Colocado **junto** ao theme toggle:
  - Root: `Layout.tsx` (canto inferior direito — empilhar ou agrupar com gap).
  - Lab: `LabPage.tsx` (mesmo canto; agrupar theme + locale).

### Conteúdo a migrar

| Área | Arquivos | Notas |
|------|----------|--------|
| Root | `ProfileCard.tsx` | Todo o copy; links e e-mail ficam iguais |
| Lab pitch | `sharedPitch.ts` → passa a expor só dados não-traduzíveis (name, email, urls); textos via `t()` | Keywords traduzíveis onde fizer sentido (ex. "Produtos Digitais" → "Digital Products") |
| Lab hero CTAs | `PitchCopy.tsx` | Âncoras `#memphis-more` / `#memphis-contact` |
| Lab works | `MemphisMore.tsx` | Títulos de projeto podem ficar em inglês técnico; blurbs/tags/intro traduzir |
| Lab contact | `MemphisContact.tsx` | Headings, body, labels; handles (`@silash00`) e URLs não traduzem |
| Easter eggs | `MemphisPop.tsx` `EGG_COPY` | Traduzir |
| A11y labels | toggles, `aria-label` de ícones se visíveis | Onde houver string de UI |

**Não traduzir:** nome próprio, e-mail, URLs, handles, nomes de libs (`React`, `n8n`, `RAG` como sigla), ids de seção/DOM.

### Scroll suave

Já existe via CSS em `html:has(.lab-memphis)`. Sem mudança necessária para i18n.

## UX

1. Usuário abre o site → idioma inicial do browser (ou `localStorage` se já escolheu).
2. Clica PT/EN → copy troca na hora; `lang` no `<html>` atualiza; preferência salva.
3. Theme e locale são controles independentes.

## Out of scope

- Locale na URL (`/en`, `?lang=`)
- Detecção por IP / geo
- Mais de dois locales
- Pluralização / ICU / rich text no `t()`
- Tradução de assets/imagens
- SSR / meta tags por idioma (site é SPA estático)

## Acceptance criteria

- [ ] Toggle PT/EN visível no root e no `/lab`, próximo ao theme toggle
- [ ] Todo copy de UI do root e do lab respeita o locale ativo
- [ ] Preferência persiste após reload (após toggle)
- [ ] Primeira visita sem `localStorage` segue `navigator.language`
- [ ] `<html lang>` reflete o locale
- [ ] Build (`tsc` + vite) e lint passam
- [ ] Sem dependência nova de i18n no `package.json`

## Implementation notes

- Manter paridade de chaves entre `pt-BR.ts` e `en-US.ts` (TypeScript deve falhar se faltar chave).
- Preferir `satisfies Messages` ou tipo derivado de `pt-BR` como source of truth.
- Não misturar strings hardcoded novas após a migração.
